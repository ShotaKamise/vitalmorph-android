# VitaMorph 実装計画

> Phase 1〜6は完了(v0.9)。本書はその履歴として保持する。
> 以降の拡張(71体進化表、性格、大会状態の永続化、図鑑、レシピ等)と完成(v1.0)までの計画・進捗は
> `docs/COMPLETION_PLAN.md` と `docs/HANDOFF_STATUS.md` を参照。

## 目標

健康管理を続けることでモンスターが成長し、交流、会話、ミニゲーム、戦術バトル、生まれ変わりを1年間楽しめるAndroidアプリにする。

## Phase 1: v0.4 データ基盤、名前、性別

- Room導入
- 既存SharedPreferencesを維持しながら段階的に移行
- トレーナー名の初回設定と設定画面からの変更
- `MonsterSex.MALE` / `MonsterSex.FEMALE`
- 孵化時に約50%でランダム決定し、28日間固定
- 再起動後も性別と現在世代を維持
- 次の生まれ変わりでは性別を再抽選
- 性別による能力差は付けない
- 旧ユーザーのXP、シーズン数、目標、開始日、大会ポイントを保持

完了条件:

- 既存保存データから正常に起動できる
- 同じ世代で性別が変化しない
- トレーナー名が会話用コンテキストから参照できる

## Phase 2: v0.5 新進化表、交流、会話

- 共通生命卵1枠、既存動物系14体、新規人型28体の43枠へ移行
- 第1週は性別別の幼生
- 第2週は性別別の動物系成長体3種
- 第3週は性別別の人型7職業
- 第4週は同職業の上位人型、または動物系最終形態
- タッチ部位、連続タッチ、クールダウン
- 機嫌と絆
- 時刻、食事、運動、機嫌、進化段階、大会結果を使うローカル会話
- 会話文へトレーナー名を安全に挿入
- 会話選択肢による機嫌・絆変化

会話は当初サーバーや生成AIを使わず、端末内テンプレートで実装する。

## Phase 3: v0.6 ミニゲーム、機嫌バトル連携

- コアキャッチ
- パルストレーニング
- ミールバランス
- 1回60から90秒
- 報酬は1日3回まで
- タッチ連打や時刻変更による報酬乱用対策
- 機嫌のバトル影響は最大プラスマイナス5%
- 高い絆では大会中1回だけ使える応援効果

機嫌の基準:

- 80から100: 絶好調。開始時に小さなシールド
- 60から79: 好調。素早さ微増
- 40から59: 通常
- 20から39: 不調。素早さ微減
- 0から19: 開始エネルギーに小さな不利

機嫌だけでバトル参加を禁止しない。

## Phase 4: v0.7 生まれ変わり、能力継承

- 28日終了時に世代を記録
- その世代の食事、運動、交流、大会から最大3継承ポイント
- HP、攻撃、防御、素早さへ配分
- 1ポイントは基礎能力約1%
- 1能力の継承上限は15%
- 継承能力は減らない
- 機嫌は世代ごとに初期化
- 系譜画面で過去の性別、最終形態、大会成績、継承内容を表示

## Phase 5: v0.8 食事管理MVP

- 朝食、昼食、夕食、間食
- 食品名、量、カロリー、たんぱく質、脂質、炭水化物
- 自作食品
- お気に入り、履歴、前日のコピー
- 日次、週次グラフ
- 目標との差
- 文部科学省の日本食品標準成分表を候補データとして利用条件を確認して導入
- VitaMorphが登録したNutritionRecordをHealth Connectへ書き込む
- Client Record IDとData Originを利用

栄養データの優先元をユーザーが選ぶ。

- あすけん優先
- VitaMorph優先
- 日ごとに選択

異なるアプリの栄養記録を無条件に合算しない。

## Phase 6: v0.9 食品機能拡張

- バーコードスキャン
- Open Food Factsなどの外部食品候補
- 結果確認と手動修正
- レシピ登録
- ビタミン、ミネラル
- 写真候補機能は、プライバシーと通信費を別途確認してから判断

外部通信を追加するPhaseでは、AndroidManifestのネットワーク権限、プライバシー説明、障害時のオフライン動作を実装する。
