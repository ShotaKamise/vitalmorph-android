# VitaMorph v1.1 対応計画

2026-07-12策定。ユーザーの実機フィードバック12件(Issue #2〜#13)への対応。
計画・方針はメインセッション(Fable)、実装はOpusサブエージェント。運用ルールは docs/COMPLETION_PLAN.md と同じ
(1ループ1タスク、テスト追加、`./gradlew test assembleDebug lintDebug` 成功、mainへプッシュ、本書更新、対応イシューをクローズ)。

## タスク一覧(実施順)

### U1: オンボーディング入力値の保持 (#2) — [x] 完了(2026-07-12)

- OnboardingScreenの全フォーム状態(目標6項目+デモチェック)を `rememberSaveable` へ。

### U2: 食事入力機能の削除・あすけん一本化 (#8) — [ ] 未着手

- 食事タブ・MealsScreen.kt・BarcodeScannerScreen.kt・OpenFoodFactsRepository.kt を削除。
- GameViewModelから食事入力・外部検索・レシピ・優先元関連の状態/関数を削除。栄養はHC読み取りのみへ(NutritionResolverのマージは不要になるが、関数と過去テストは削除)。
- INTERNET / CAMERA / WRITE_NUTRITION 権限、CameraX / ML Kit 依存を削除。HealthConnectRepositoryの書き込み関数と書き込み権限を削除(読み取りの自Origin除外は無害なので残してよい)。
- Roomのエンティティ・DAO・マイグレーション(v4/v5/v9含む)は既存データ保護のため**すべて保持**。FoodRepositoryは未使用化するが削除しない(将来の再有効化に備え@Suppress等でlint回避)。
- 設定タブ: 優先元セクション削除、プライバシー文を「外部通信なし・読み取りのみ」へ戻す。出典表記(FoodCatalog)はカタログ休止に伴い削除。
- 食事系テストは仕様が消えるもの(UI依存)を削除し、ドメイン(NutritionResolver等)は保持テーブルと同様に残す。
- ドキュメント(README・HANDOFF)を読み取り専用方針へ更新。

### U3: 会話テキストの品質向上 (#4) — [ ] 未着手

- テスト: 全セットで選択肢と返答の対応が定義されていること、プレースホルダ残置なし。

### U8: ターン制の交互攻撃演出 (#10) — [ ] 未着手

- BattleEngineがターンイベント列(actor / 行動名 / ダメージ / 回復 / ガード / 撃破)を返す。resolveTurnの計算結果と完全一致すること(テストで担保)。
- UIはイベントを順再生(各800ms程度): 行動者のモーション→相手のHIT+ダメージ数字→次イベント。再生中はコマンド無効。

### U9: 週末大会制+週次スケーリング (#12, #11) — [ ] 未着手

- 大会はシーズン日7/14/21/28のみ。週ごとに1回(参加週フラグをSharedPreferences永続化、シーズン完了でリセット)。
- 平日はカウントダウン+今週のトレーニング成果表示。
- 相手はプレイヤーと同ステージのフォーム(週1=成長体級の弱体、週2=成長体、週3=成熟体、週4=最終形態)から選出し、姿もそのフォームを表示。能力は週係数(0.6/0.8/1.0/1.15目安)でスケール。
- 大会ポイント: 週ごとの順位ポイント(最大10)を記録し、シーズンXP寄与は4週平均(既存スケール維持)。表示は「今週の結果」+「累計」。
- 既存の再挑戦ボタンは同週内では練習試合(ポイント加算なし)として残す。
